# Backlog Protocol

## Location

Default path: `.claude/skill-improvement/opportunities.jsonl`

Use JSON Lines so entries append cleanly and survive partial edits.

## Entry Fields

- `id`: Stable short identifier generated by the logging script.
- `created_at`: UTC timestamp.
- `updated_at`: UTC timestamp.
- `status`: `queued`, `applied`, `rejected`, or `deferred`.
- `skill`: Existing skill folder name, `NEW_SKILL`, or `UNKNOWN`.
- `kind`: `skill-update`, `new-skill`, `script`, `reference`, `validation`, or `routing`.
- `summary`: One-sentence observed opportunity.
- `evidence`: Concrete session observation that justifies the entry.
- `recommendation`: Smallest useful change or suggested new skill.
- `impact`: `low`, `medium`, or `high`.
- `confidence`: Number from `0` to `1`.
- `source_session`: Optional human-readable session label.
- `notes`: Maintenance notes added while applying or deferring.

## Batch Rules

During scheduled implementation sessions:

1. Sort queued entries by impact, then confidence, then age.
2. Batch by `skill`.
3. Apply at most three unrelated improvements per session unless the user asks for a larger batch.
4. Validate every modified skill with `quick_validate.py`.
5. Mark each handled entry as `applied`, `deferred`, or `rejected`.

## New Skill Recommendations

Use `skill: "NEW_SKILL"` and `kind: "new-skill"`.

The `recommendation` should include:

- Proposed skill name.
- Triggering contexts.
- Reusable resources likely needed.
- Why existing skills should not absorb it.
